Week 01

Notes:

  • WDD 330 is designed to teach students the skills to create web applications using HTML, CSS, and JavaScript. Students should do this with plain/pure languages, without any third-party framework, libraries, or plugins.
  • In this course, the team exists to evaluate my own learning not the team's.

  • Github Pages is a static site hosting service that takes HTML, CSS, and JavaScript files straight from a repository on GitHub, optionally runs the files through a build process, and publishes a website. You can see examples of GitHub Pages sites in the GitHub Pages examples collection. ("About GitHub Pages" in Github Docs)
  • The list used in main.js was an array of objects. It allows us to store multiple data for an object of the list.
  • When referencing to an object in an array, we can use "index.name" format.
  • We can use <pre> and <code> tags when we want to include code snippets in our webpages.
  • We can also set a scroll bar visible when needed using "overflow-x:auto;"

  • UX is the most important thing.
  • The ultimate word for the good UX is simple. Keep it simple but useful, and helpful. When there is usability, true beauty comes into play.
  • Think about how users use it. Place buttons at the places that are easy to reach. Usually at the bottom.
  • Speed is also important. Optimize the size of files used in the web page. Using web fonts, svg files, and reducing and minimizing the file sizes are always good practices.
