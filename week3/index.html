Notes:

Chapter 5

  • Object in JavaScript is a self contained set of related values and functions
  • Objects act as "a collection of named properties that map to any JavaScript value such as strings, numbers, booleans, arrays and functions.
  • A method is when a property’s value is a function
  • Objects are used to keep any related info and functionality together
  • Object literals example: 
    const superman = {
                        name: 'Superman',
                        'real name': 'Clark Kent',
                        height: 75,
                        weight: 235,
                        hero: true,
                        villain: false,
                        allies: ['Batman','Supergirl','Superboy'],
                        fly() {
                            return 'Up, up and away!';
                        }
                    };
  • The ability to create objects with computed property keys was introduced in ES6. This means that JavaScript code can be placed inside square brackets and the property key will be the return value of that code. 
    
                            const hulk = { name: 'Hulk', ['catch' + 'Phrase']: 'Hulk Smash!' };
  • includes() can be used on a string to see if a certain character exists in the string
  • Template literals are written with backticks (`) and JS expressions can be inserted within a template literal
  • Symbols can be used to create unique values and avoid duplication
  • Numbers can also be represented in octal, hexadecimal, and exponential notation
  • Variable values can be reassigned using = , but can also be incremented by += and decremented by -=
  • Infinity is an error value that means the number is too big to be deal with in JS
  • Strings and numbers can be converted into each other using parseInt() and toString()
  • Undefined means value is missing, while null means “no value” and acts like a 0
  • Bitwise operators (& for AND and | for OR) work with numbers that are written in binary
  • Soft equality (==) will try to convert and match the data types automatically, while hard equality (===) will check the values as their original data types

Chapter 6

  • Arrays are lists of values and can be created by [] or the Array constructor
  • Values can be added by assigning them to different indices within the array
  • You can have different types in one array (like string, numbers, other arrays, etc)
  • Values can be deleted using the “delete” operator
  • Destructuring means to take values out of the array to use separately
  • Arrays have a length property that shows how many values are in the array
  • pop() removes the last item, shift() removes the first item, push() adds a new value to the end, and unshift() adds a value to the start
  • concat() or a spread operator (...) can be used to merge one or more arrays together
  • join() can be used to convert an array into a string, and you can specify what character to join the strings within the array with
  • slice() can create an array from another array, and splice() can replace a value in an array with another
  • There are multiple ways to find if a value is in an array, like indexOf() and includes()
  • Arrays can be multidimensional, like a matrix
  • A Set is like an array but it cannot contain duplicate values, only unique values
  • Memory leaks happen when a variable that isn’t being used is still stored in memory, but modern languages like JS have garbage collection which automatically removes unused values
  • Maps can be used to keep a list of key and value pairs, keys need to be strings and can be used to refer to the value stored with them
  • get() can be used to look up a value by key, has() checks if the value exists, delete() removes the key value pair, and set() will add a key value pair
  • If/else statements can be used to run code only if a certain condition is met
  • Ternary operators and switch statements fulfill similar functions but with different syntax and use cases
  • While and for loops let us run code repeatedly until a certain condition is met (or infinitely, for an infinite loop)
  • For loops are good for looping through arrays due to the syntax of the for loop

Chapter 7

  • Functions can be declared using the “function” key word and with a name
  • Anonymous functions don’t have names and can be assigned to variables
  • Functions are called by entering its name with parentheses, and either return undefined by default or whatever value the programmer set it to return
  • Parameters are set when the function is defined, while arguments are for when the function is invoked
  • Values can be passed into functions on the fly using parameters/arguments
  • You can pass as many arguments as you want into a function, and the rest operator (...) can put all arguments into an array to be used within the function
  • Default parameters can be set within the function declaration using the equal sign
  • Arrow functions (const square = x => x*x) are simpler ways to create anonymous functions
  • Hoisting allows functions/variables to be called before they’re defined in the code
  • Callbacks allow other functions to be passed as arguments into a function, which can be called within that function
  • forEach(), map(), reduce(), and filter() are some ways to iterate through arrays

            quiz-ninja.js

            // Set the questions
            const quiz = [
                ["What is the name of world's highest mountain","Mount Everest"],
                ["Who is the current president of the United States?","Joe Biden"],
                ["What is the name of world's longest river?","Nile"]
            ];

            function start(quiz){
            let score = 0;

            // main loop
            for(const [question,answer] of quiz){
                const response = ask(question);
                check(response,answer);
            }
            // end of main game loop
            gameOver();

            // ask and check
            function ask(question){
                return prompt(question);
            }

            function check(response,answer){
                // check if answer is correct
                if(response === answer){
                alert('Correct!');
                // increase score by 1
                score++;
                } else {
                alert(`Wrong. The correct answer was ${answer}`);
                }
            }

            function gameOver(){
                // Report the player's score
                alert(`Game Over, you scored ${score} point${score !== 1 ? 's' : ''}`);
            }
            }
