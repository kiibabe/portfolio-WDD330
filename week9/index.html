Chapter 9: The Window Object

  • BOM (Browser Object Model) contains info about the browser and computer screen
  • The window object is a global object in a browser environment
  • Alert dialog messages can be invoked through the window object
  • window.navigator property has info about what browser is being used to run the code
  • window.location has info about the current URL and the different properties within it can break up the URL into pieces (like host, protocol, and port)
  • window.history has info about previously accessed pages in the current browser session
  • The window object has methods to open, close and control the browser window size
  • Screen information can be received from the window.screen property (like height, width)
  • The document object, contained in the window object, has to do with the page itself like the content and cookies
  • setTimeout() and setInterval() are two methods that let the user time certain functions and add delays to code
  • Those methods can be used to create animations by modifying the css based on the timer

Chapter 14: HTML5 APIs

  • HTML5 is the latest version of HTML and has new features that make development easier
  • The data-attribute can be used to insert data in a page that is ignored by the browser
  • The data can be accessed via the dataset property in each element
  • When referencing data in the dataset property, the data- prefix is dropped (for example, data-dropdown would be accessed like such: element.dataset.dropdown)
  • Localstorage and sessionstorage can be used to store data locally in the browser
  • Storage events are fired each time changes are made to the web storage
  • Geolocation API can get the geographical location of the device accessing the page, including coordinates, speed, altitude, and heading
  • Web workers allow users to run multiple processes in the background, like a thread
  • Shared web workers can be used by multiple scripts on the same domain, and service web workers can intercept network requests
  • Websockets allows for push messaging, where the connection between client and server is kept open
  • Notifications can be controlled, as well as asking for and granting permissions to display notifications
  • Multimedia (like video and audio) can be added easily via <video> and <audio> tags
  • The canvas element allows for graphics like lines and shapes to be drawn in it
  • Shims/polyfills allow older browsers to also use the new APIs without issues by filling in missing code automatically without the user needing to code anything extra
