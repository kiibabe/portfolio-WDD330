Chapter 11: Further Functions

  • In JavaScript, functions are first-class objects, so they can be passed around just like other values
  • Functions can have their own properties and methods
  • call() & apply() methods: They can invoke a function with the value of "this" bound to an object that is provided as an argument. Useful if the data that is being used as an argument is already in the form of an array
  • cache property: When a function takes a long time to compute a return value, we can save the result in cache property. It enables us to return the saved value, rather than computing it again
  • Immediately Invoked Function Expression: IIFE. An anonymous function. 
      (function(){
        const name = 'Kisetsu';
        console.log(`Hello ${name}`);
    })();
  • Safe use of "strict" mode 
      (function() {
        'use strict';
                        
        // All your code would go inside this function
                        
    })();
  • Functions are able to dynamically redefine themselves in the body of the function, depending on certain conditions
  • Recursive function: This keeps invoking itself until the designated condition is met. Useful when iterative processes are involved
  • JavaScript is a single-threaded environment
  • Callback: Provided as an argument to another function
  • Callback hell: A common issue. Tangled mess of code
  • Promises: Represents the future result of an asynchronous operation. This can be used instead of callbacks to deal with multiple asynchronous actions in sequence. They also provide a nicer mechanism for handling errors
  • Creating a promise: 
      const promise = new Promise( (resolve, reject) => {
        // initialization code goes here
        if (success) {
            esolve(value);
        } else {
            reject(error);
        }
    });
  • Functions that return other functions: Higher-order functions
  • Generator: Put an asterisk (*) after the function keyword. This return an iterator object that provides a next() method
  • Pure functions: Do not rely on the state of the code, have no side-effects, and always give the same result when given the same arguments
  • Closure: One of JavaScriptâ€™s most powerful features. A reference to a variable that was created inside the scope of another function, but is then kept alive and used in another part of the program

Chapter 13: AJAX

  • Ajax allows web pages to communicate asynchronously with a server, and it dynamically updates without reloading
  • API: An application programming interface: A collection of methods that allows external access to another program or service
  • Ajax can be used for communicating with external APIs
  • The Fetch API: fetch() method
  • The data is usually sent in the form of JSON
