Chapter 8: Forms

  • Forms: a common method of interacting with a web page 
         <!DOCTYPE html>
        <html lang='en'>
        <head>
            <meta charset='utf-8'>
            <title>Search</title>
        </head>
        <body>
            <form name='search' action='/search'>
                <input name='queryInput'>
                <button type='submit'>Search</button>
            </form>
            <script src='javascript.js'></script>
        </body>
        </html>
  • Accessing Form Elements: 
        const form = document.forms[0];
    const form = document.getElementsByTagname('form')[0];
    const form = document.forms.search;
    const form = document.forms['search'];
  • Forms UX:
    • Use correct and semantic markup
    • Make forms keyboard accessible and using WAI-AIRA labels
  • We need to be careful when using reset buttons because it is generally considered poor UX for it's too easy to click and delete what has been entered,
  • Form Events:
    • focus: when an element is focused on
    • blur: when the user moves the focus away
    • change: when the user moves the focus away after changing it
  • We can stop the form from being submitted to that URL altogether by using the "preventDefault()" method
  • Common form controls:
    • <input>
    • <select>
    • <textarea>
    • <button>
  • New Attributes for input in HTML5:
    • autofocus: Gives focus to this element when a page loads. Equivalent to "document.forms.hero.heroName.focus();"
    • placeholder: Inserts the value provided in the input field until the user enters some text
    • maxlength: Limit the number of characters in a field
  • Hidden fields can be made using input fields with "type='hidden'". It is often used to send information like settings or the information that the user has already provided. It is visible in HTML so we cannot deal with sensitive data with this method.
  • Form validation: The process of checking whether a user has entered the information into a form correctly
    • Required fields are filled
    • Email address is valid
    • Numbers entered where numerical data is required
    • A password is at least at the minimum length
  • HTML5 has its own validation API that can be used, although it is not supported from all browsers now. The error messages that it produces can look inconsistent across browsers and are difficult to style

Chapter 9: The Window Object

  • The window object is the global object in a browser.
  • The Browser Object Model (or BOM for short) is a collection of properties and methods that contain information about the browser and computer screen. It only makes sense in a browser environment.
  • In a browser environment, the global object is the　”window　object”
  • window.navigator: Gives information about the user’s browser and operating system (but it's unrealiable)
  • window.location: Provides information about the URL of the current page. It takes href, protocol, host, port, pathname, origin, hash, and serach properties.
  • window.history: Keeps information about the pages that have been visited in the session. There is length property and go(), forward(), and back() methods.
  • window.screen: Provides information about the user’s screen
  • document.write() should be avoided
  • window.setInterval(): Works in a similar way to window.setTimeout(), except that it will repeatedly invoke the callback function after every given number of milliseconds
  • document.cookie: Lets us store pieces of information between requests
  • window.setTimeout(): Can be used to call a function after a set amount of time. clearTimeout() can cancel it
  • window.setInterval(): Can be used to call a function repeatedly. clearInterval() can stop it.
  • window.requestAnimationFrame(): Produces smooth/optimized animation by using the browser's built-in graphics capabilities. cancelAnimationFrame() can cancel it.

Chapter 10: Testing and Debugging

  • Types of error
    • System error: There is problem with the system or external devices with which the program is interacting
    • Programmer error: The program contains incorrect syntax or faulty logic (It could even be as simple as a typo)
    • User error: The user has entered data incorrectly, which the program is unable to handle
  • Exception: An error that produces a return value that can then be used by the program to deal with the error
  • Stack Trace, which an exception will also produce can help us debug. For example, it can help us find where an error originates from.
  • We need to ensure that the code we write fails loudly because failing silently makes errors difficult to spot and longer to track down.
  • Strict mode encourages a better quality of JavaScript that befits a programmer, so its use is recommended. It requires the following code to be added at the beginning of a javascript file: 
         'use strict';
  • "Linting tools" like JS Lint, JS Hint, andES Lint can be used to test the quality of code
  • "alert()" stops a program from running until OK is clicked, so it allows us to effectively put breakpoints in the code that allows us to check the value of variables at that point
  • console.trace(): Logs an interactive stack trace in the console. This will show the functions that were called in the lead up to an exception occurring while the code is running
  • Exceptions can be thrown using the "throw" statement
  • Any code placed inside a "try" block will pass any error objects to a "catch" block when an exception occurs. Any code inside a "finally" block will run if an exception does or does not occur
  • Test-driven development: The practice of writing tests that fail, then writing the code that passes the test, then refactoring the code every time a new feature is implemented

Chapter 12: Object-Oriented Programming in JavaScript

  • "Object-oriented programming (OOP) is a way of programming that uses objects that encapsulate their own properties and methods"
  • The main concepts of OOP:
    • Encapsulation
    • Polymorphism
    • Inheritance
  • The keyword "this" refers to the object returned by the function inside a constructor function or class declaration
  • All instances of a class or constructor function inherit all the properties and methods of its prototype
  • Private properties and methods can be created by defining variables usingconstand defining a function inside a constructor function
  • Getter and Setter functions can make private properties and methods public.
  • Monkey-patching: The process of adding methods to built-in objects by augmenting their prototypes. We need to be careful with this because it can cause unexpected behavior in the way built-in objects work
  • Polymorphism allows objects to override common methods with a more specific implementation
  • Methods can be borrowed from other objects

Chapter 15: Modern JavaScript Development

  • JavaScript libraries provide methods to make common tasks easier
  • Although libraries can make programming easier, we need to be carerful with whether we need a library or what library to choose
  • "jQuery" and "Lodash" are the popular libraries, and "npm" and "Yarn2 are package managers that can be used to install JavaScript packages
  • Module: a self-contained piece of code that provides functions and methods that can then be used in other files and by other modules
  • The MVC pattern: This is used to organize code into distinct sections that are responsible for different elements of an application
  • Template files can separate view code from javascript code. They also enable dynamic code and programming logic
  • "React" and "Vue.js" are popular JavaScript view libraries
  • Minification: The process of removing any redundant letters from the code
